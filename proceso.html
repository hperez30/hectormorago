Proceso

Published: lun 06 abril 2020
Updated: lun 06 abril 2020
By Hector Perez-Morago

In blog.

tags: SSOO teoria otros

Un proceso es una instancia de un programa en ejecución. En un instante determinado puede haber varias instancias (réplicas) de un mismo programa en ejecución.

Los procesos son independientes, aunque sean instancias de un mismo programa.

Un proceso se compone de varios elementos, es decir, no es solamente el código del programa, también incluye la actividad actual como:

  • Contador de programa
  • Registros del procesador
  • Imagen de memoria

Un proceso puede pasar por varios estados distintos desde su creación hasta su finalización. Es lo que se conoce como ciclo de vida de un proceso. El número de estados depende del SO en cuestión pero entre :

  • Nuevo: el proceso está siendo creado.
  • Listo: está a la espera de que le asignen un procesador.
  • En ejecución: se están ejecutando instrucciones del proceso.
  • Bloqueado: esperando que ocurra algún suceso (E/S, comunicación, sincronización . . .) que lo desbloquee.
  • Terminado.
  • Otros estados:
  • Preparado: el proceso está a la espera para ser ejecutado cuando tenga suficientes recursos (ejemplo: ejecución por lotes).
  • Suspendido: no puede pasar a ejecución porque reside enteramente en la memoria de intercambio. El objetivo de suspender procesos es el de reducir el grado de multiprogramación (número de procesos que instante determinado se encuentran cargados en memoria principal).

Cada proceso se representa dentro del SO con un bloque de control de proceso o PCB. El PCB contiene información asociada al proceso, como:

  • Identificador del proceso (pid) y del padre (ppid)
  • Identificador de usuario (uid) y de grupo (gid)
  • Estado del proceso: listo, bloqueado, ...
  • Estado del procesador: valores de los registros (incluyendo contador de programa). Esta información se guarda cuando se produce una interrrupción.
  • Información contable: tiempo empleado, límites, ...
  • Información de planificación: prioridad, ...
  • Información de gestión de memoria: descripción de los segmentos, tabla de páginas,...
  • Información del estado de E/S: disp. asignados, lista de descriptores de ficheros abiertos,...
  • Temporizadores, señales, semáforos, puertos,...

El SO tiene que mantner en todo momento una serie de estructuras de datos y tablas entre las que podemos citar:

  • Tabla de Procesos -> Tabla de PCBs, donde cada elemento de la tabla es el PCB de un proceso
  • Tabla de ficheros abiertos del sistema, cada proceso tiene su tabla de descriptores de ficheros, con apuntadores a entradas de esta tabla
  • Tabla de Memoria
  • Tabla de E/S, almacena operaciones de E/S pendientes

Creación de procesos

Los procesos pueden crear nuevos procesos. El proceso creador se denomina padre y el proceso creado se denomina hijo. A su vez los hijos también pueden crear nuevos procesos, . . ., formándose un árbol de procesos. La mayoría de los sistemas operativos utilizan un identificador de proceso (pid) unívoco para cada proceso.
