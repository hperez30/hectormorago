Efficient Identification of Core and Dead Features in Variability Models

Context

Variability models play a key role in product line engineering as they are used to represent the common and variable features that products may include, and what constraints among the features must be satisfied to guarantee the validity of the products. Valuable analysis operations on variability models can be performed by black box reusing logic engines, such as SAT-solvers and binary decision diagram libraries. Unfortunately, such kind of reuse implies long computation times for operations that need calling the engines many times. To overcome this problem, we work on new algorithms that directly deal with the data structure of a binary decision diagram encoding a variability model. In particular, the algorithm published in this site is specifically designed to detect core and dead features

Description

This repository provides an algorithm to identify core and dead features, which is written in the C++ programming language as an extension of the BuDDy BDD package.

A Bibliometric Analysis of 20 Years of Research on Software Product Lines

Context

Software product line engineering has proven to be an efficient paradigm to developing families of similar software systems at lower costs, in shorter time, and with higher quality.

Description

This repository stores the raw data retrieved from ISIWoS and Scopus after performing the query: "software product line" or (("product line" or "mass customization" or "product famil" or "program family" or "software factory" or "product platform") and (("domain engineering" or "application engineering") or ("feature model" or "feature diagram" or "decision model" or "decision diagram") or (software and variabilit* and commonalit*)))

Querying users and meetings in Cisco CMS

Context

Cisco CMS is a video conferencing system developed by cisco.

Description

This repository provides an example application to interact with the Cisco CMS API Rest to create, edit and retrieve meetings and users. The application uses apiary mock server which is freely available at https://ciscocms.docs.apiary.io/ in order to test all functionalities. The application is written in PHP, HMTL, CSS and javascript.
